RENAULT NISSAN INDIA Pvt Ltd

Projects in Engine Control Algorithm Development at Nissan:

1.Development of Engine control subsystem components (Dec 2011 to Sep-2014)

Description:
Model based design of components will help in identifying issues at the earlier stages of product development life cycle. Production code is auto generated using Simulink embedded coder software.

Responsibilities:

  • Understanding the requirement using timing diagrams
  • Development of control models using Nissan Powertrain guidelines.
  • Develop open loop test cases using Matlab editor.
  • Perform automated model checks (e.g. datatype, Naming conventions) using Matlab scripts for Nissan Powertrain guidelines.
  • Perform model checks for Integrator without protection(overflow) and division by zero errors.
  • Perform model coverage study using Reactis tool.
  • Perform unit validation of requirement using open loop test simulation.
2.OBD: Coolant Temperature Sensor Stuck diagnosis improvement (Mar 2011 to Sep-2011)

Description:
Coolant temperature sensor information is critical in determining the state of engine and accordingly necessary ignition, air intake and fuel injection corrections are taken to improve the performance. If the coolant temperature sensor value does not change its value for a long time, then the coolant sensor is considered stuck. OBD II system should have necessary monitoring method to observe this stuck condition and raise Engine MIL lamp.  

3.OBD: Purge Flow Control (Oct 2011 to Sep 2014)

Description:
Automobile fuel system is the major source of hydrocarbon vapor emissions. The purpose of the automobile evaporative emission systems is to reduce or eliminate the release of vaporized hydrocarbons (HC) into the atmosphere.  With O2 sensor feedback the canister purge solenoid can be activated when the vehicle is capable of running lean, most commonly at warm engine cruise. Canister drain cut valve needs to be operated accurately in order to avoid damage of fuel system due to negative pressure.
Tools: Matlab/Simulink, Reactis and dSPACE.

Responsibilities:
  • Understood the OBD II requirement from CARB (California Air Resource Board) regulation document.
  • Identified the current algorithm issue by analyzing the test field data.
  • Applied FTA (Fault Tree Analysis) and DA (Decision Analysis) techniques and proposed the counter measure for the current problem.
  • Verified the counter measure logic using dSPACE HILS Rapid control Prototyping technique using dSPACE Microautobox and GSI configuration tools.
  • Performed FMEA (Failure Mode Effect Analysis) technique to analyze the influence of counter measure on other control functions and engine performance.
  • Applied QFD (Quality Function Deployment) method to check the implementation of all the requirement functions in the control logic.
  • Updating the requirement document and developing the model for the required changes according to model-based design guidelines.
  • Preparation of test case and Validation of Counter measure in HILS test bench.
  • Preparation of calibration method for vehicle calibration.
